## Risk Log — Intentionally Retained Code

### MarketEventSystem (retained, not wired)

**Files:** `game/scripts/systems/market_event_system.gd`, `game/resources/market_event_definition.gd`, `game/content/events/market_events.json`

**Rationale:** MarketEventSystem is a fully implemented system (366 lines) with save/load support, event lifecycle management, weighted selection, and trend multiplier calculation. It is documented in the project brief as a core feature (15% daily event chance, boom/bust/spike/shift types). The DataLoader already parses `market_events.json` into `MarketEventDefinition` resources.

**What was cleaned:** All integration points in other systems were removed (EconomySystem setter, AudioManager handler, EventBus signals). The system is now fully self-contained with zero coupling to the rest of the codebase.

**To wire up:** Add instantiation in `game_world.gd:_setup_systems()`, restore EventBus signals, call `economy_system.set_market_event_system()`, and reconnect AudioManager.

### DataLoader market event methods (retained)

**Functions:** `get_market_event()`, `get_all_market_events()`, `get_market_events_for_store()`, `_load_market_events()`

**Rationale:** These are part of the DataLoader's content registry and are called by MarketEventSystem. Removing them would require also deleting MarketEventSystem (which is a complete feature). They load valid JSON content that is present in the repo.
