# Documentation Consolidation Audit

Date: 2026-04-17

## Goal

Bring the active documentation set in line with the current Godot/GDScript
codebase, keep `README.md` as the only root project document, and document only
behavior that could be verified from repository files.

## Added or Rewritten

- `README.md` - tightened project overview, local run instructions, test entry
  point, deployment basics, and docs links.
- `docs/index.md` - clarified what counts as active project docs versus
  configuration or template Markdown elsewhere in the repo.
- `docs/setup.md` - updated Godot version guidance and documented the shared
  CLI wrapper script.
- `docs/architecture.md` - expanded verified `GameWorld` UI composition and
  clarified save/load coverage.
- `docs/content-data.md` - corrected content-tree layout, compatibility files,
  skip rules, and non-Resource config data.
- `docs/testing.md` - corrected CI behavior to reflect the active `gut-tests`
  job and documented the current workflow/docs mismatch around `CLAUDE.md`.
- `docs/configuration-deployment.md` - updated the validate workflow section to
  match the checked-in GitHub Actions jobs.
- `docs/contributing.md` - added `.editorconfig` formatting rules and kept the
  root/docs documentation boundary explicit.
- `docs/audits/docs-consolidation.md` - rewritten to describe this pass
  accurately.

## Deleted

- `AIDLC_FUTURES.md` - autogenerated run metadata removed from the repository
  root so `README.md` is the only root project doc.

## Consolidated Topics

- The active docs set remains `README.md` plus the focused docs under `docs/`.
- Setup, test, CI, and export details were aligned so each topic has one
  canonical home instead of partial overlap across multiple pages.
- Content-tree documentation now distinguishes canonical subdirectories from
  compatibility-era root JSON files that still load today.
- Tooling and template Markdown under `.github/` and `tools/` were left in
  place because they are configuration or scaffolding, not active game docs.

## Notes

This pass did not need another large tree-wide consolidation. The repository
already had a lean active docs set under `docs/`; the main work was correcting
specific inaccuracies, documenting current compatibility behavior, and removing
the one stray autogenerated root Markdown file.
