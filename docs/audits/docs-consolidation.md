## Intentional gaps

- **No reference docs were collapsed or split this pass.** The current
  layout (`README.md` at root + nine top-level docs in `docs/` + nested
  `architecture/ownership.md`, `style/visual-grammar.md`, and
  `audits/`) already matches the structure rule in this prompt: README is
  lean, everything else lives under `docs/`. Nothing to merge or move.
- **`BRAINDUMP.md` left untouched.** Customer voice — out of scope for any
  documentation pass per the project rule embedded in `docs/contributing.md`
  and `docs/index.md` ("`README.md` is the only active project doc at the
  repository root. `BRAINDUMP.md` is the customer-voice state assessment
  and is not edited by documentation passes.").
- **Audit reports under `docs/audits/` left untouched.** Each is a
  point-in-time record of a code-quality pass (cleanup, error handling,
  security, SSOT). Rewriting them would lose the historical signal they
  provide; deleting them would break the cross-references in
  `docs/index.md` and `docs/roadmap.md` Phase 0.1.
- **`AuditLog.pass_check(&"boot_scene_ready", …)` not added to
  `docs/architecture.md` §"Boot Flow".** It runs between the `boot_completed`
  emit and the MAIN_MENU transition, but it is an instrumentation artifact
  that appears in audit logs only — not part of the player-visible boot
  contract. Documenting it would invite drift each time we add or remove
  audit checkpoints.
- **`docs/audits/2026-05-01-audit.md` left as-is.** It is regenerated by
  `tests/audit_run.sh` (locally and in CI) and is intentionally a snapshot.
  This pass corrects only the *index.md pointer* to it, not the file.
- **Roadmap kept as-is.** `docs/roadmap.md` is the multi-phase finalization
  plan. The "Current state" paragraph and Phase 0.1 are accurate
  (5-store roster, three SSOT tripwires checked in and wired into
  `tests/run_tests.sh`). Future-phase prose intentionally describes work
  not yet shipped — that is the doc's purpose, not a verifiability gap.
