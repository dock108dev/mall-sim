# Documentation Consolidation Audit

Date: 2026-04-18

## Goal

Rebuild the active documentation set from the repository's current Godot and
GDScript source of truth, keep `README.md` as the only root project document,
and leave only useful supporting docs under `docs/`.

## Rewritten

- `README.md` - refreshed the project overview, local run flow, test runner
  behavior, deployment basics, and docs links.
- `docs/index.md` - clarified the active docs boundary and separated core docs
  from point-in-time audit notes.
- `docs/setup.md` - aligned setup guidance with the current helper scripts,
  Godot resolution order, and actual test pipeline.
- `docs/architecture.md` - rewrote boot, menu, `GameWorld`, UI, controller
  wiring, and save/load behavior around the current runtime.
- `docs/content-data.md` - rewrote the loader pipeline, current content tree,
  canonical ID rules, scene-path constraints, typed resources, and compatibility
  behavior.
- `docs/testing.md` - corrected the local test runner steps, GUT config, shell
  validators, and current CI workflow behavior.
- `docs/configuration-deployment.md` - aligned project settings, persistence
  paths, export presets, and GitHub Actions automation with the checked-in
  files.
- `docs/contributing.md` - kept the current formatting, naming, content, test,
  and documentation-boundary guidance in one place.
- `docs/audits/docs-consolidation.md` - updated this summary for the current
  documentation rewrite.

## Deleted

- `AIDLC_FUTURES.md` - removed the autogenerated root run-summary file so the
  repository root returns to `README.md` as the only active project document.

## Consolidated

- Active documentation remains a compact set: `README.md` plus the focused docs
  in `docs/`.
- Setup, testing, CI, and deployment details were re-split so each topic has a
  clear canonical home instead of overlapping summaries.
- Content documentation now distinguishes canonical subdirectories,
  compatibility-era root JSON files, and the remaining milestone split between
  boot loading and `ProgressionSystem`.
- `docs/audits/*.md` were kept as dated audit/history notes rather than mixed
  into the active how-to/reference docs set.
- Markdown under `.github/`, `tools/`, and vendored addon directories was left
  in place because it is configuration, template, or third-party material rather
  than active project documentation.
