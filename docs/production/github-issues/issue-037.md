# Issue 037: Design and document UI/UX specification

**Wave**: wave-2
**Milestone**: M2 Core Loop Depth
**Labels**: `design`, `ui`, `ux`, `phase:m2`, `priority:medium`
**Dependencies**: issue-031

## Why This Matters

This is a management sim. UI is the primary gameplay surface.

## Scope

Create authoritative design doc for all UI panels. Layout specs for inventory, pricing, catalog, day summary, store selection, main menu. Keyboard shortcuts. Panel transition behavior.

## Deliverables

- docs/design/UI_SPEC.md
- Layout specs for each panel (position, size, content)
- Keyboard shortcut map
- Panel open/close behavior
- Tooltip system spec
- HUD layout spec
- Day summary layout
- Store selection screen spec

## Acceptance Criteria

- Every M1-M3 UI panel has a spec
- Shortcuts don't conflict
- Panel behavior is unambiguous
- Specs reference actual systems they display data from
